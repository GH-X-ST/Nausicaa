R11 Readiness Smoke
===================

Purpose
-------
`run_r11_readiness_smoke.py` is a small engineering check between a newly
frozen R10 governor and real-flight deployment. It is not final thesis or
journal validation evidence.

The smoke run preserves the official R11 structure:

- all 8 R11 ladder blocks, L0 through L7;
- all 5 library tiers;
- all 4 final policies: no memory, h3, h10, h30;
- paired start conditions across ladders, tiers, and policies;
- paired history-launch sequences;
- frozen R10 governor config consumption;
- same actuator, latency, quantisation, selector, and memory runtime path.

It reduces only the number of paired local cases per ladder.

Default Size
------------
Default `outer_cases_per_ladder = 2`.

Expected launches:

- final launches: 320
- history launches: 3440

This is intended to run in roughly 30 minutes on the current 8-worker setup,
depending on machine load and the newly regenerated primitive library.

Use
---
After R5/R7/R8/R10 have been regenerated and R10 has written a frozen governor
config, run:

```
C:\ProgramData\miniforge3\python.exe 03_Control\04_Scenarios\run_r11_readiness_smoke.py
```

Typical regenerated-evidence command:

```
C:\ProgramData\miniforge3\python.exe 03_Control\04_Scenarios\run_r11_readiness_smoke.py ^
  --library-root 03_Control\05_Results\R8_library_size_study\E01 ^
  --outcome-root 03_Control\05_Results\R8_outcome\E01 ^
  --source-w2-root 03_Control\05_Results\R5_dense\E01 ^
  --governor-config-path 03_Control\05_Results\R10_learn\E01\manifests\frozen_governor_config_for_r11.json ^
  --run-label SM01
```

For a schedule-only dry run:

```
C:\ProgramData\miniforge3\python.exe 03_Control\04_Scenarios\run_r11_readiness_smoke.py --dry-run-schedule --run-label SM_DRY
```

Interpretation
--------------
Passing this smoke run means the frozen R10/R8/R5 handoff is plausible for
real-flight engineering use. It does not support final statistical claims.

The full R11 remains the evidence stage for thesis/journal reporting.
