Definitions
-----------
- Primitive: short LQR-stabilised manoeuvre with entry checks, a fixed 0.10 s horizon, 5 controller-input slots at a 20 ms controller update period, exit checks, metrics, and failure labels.
- LQR controller: time-invariant local feedback controller embedded inside one primitive, identified by `controller_id`, `linearisation_id`, Q/R weights, gain checksum, and closed-loop audit status.
- Primitive-controller variant: the active library item combining one primitive entry set, reference state/command, finite horizon, local LQR Q/R/K, exit checks, failure labels, and claim status.
- Environment context: local flow and safety features computed from the current state and wind field.
- Episode: one simulated or physical launch from the fixed nominal launch gate.
- W0/W1 dense generation: the heavy primitive-library creation stage. W0 uses no updraft. W1 uses the simple variable-Gaussian plume surrogate for both single-fan and four-fan cases. Each primitive-controller variant receives its own LQR tuning. All solved, blocked, weak, failed, rejected, continuation-valid, and x-y terminal-useful rows are preserved.
- W2 dense survival sweep: fixed-LQR replay using the single-fan and four-fan GP-corrected annular-Gaussian surrogate (`annular-GP` in the repository), panel-wise wind, feedback latency, command timing, actuator lag, and realistic termination. No hidden LQR retuning.
- W3 dense survival sweep: fixed-LQR replay under domain randomisation of updraft parameters, fan positions, fan number, glider model parameters, feedback latency, command timing, and actuator delay. No hidden LQR retuning.
- Post-W3 library-size cross-study: heavy, balanced, light, and no-clustering/no-merging cases applied only after W3 survival replay, used to measure the impact of reducing the library rather than assuming one final compact-library size.
- Full-loop validation: frozen candidate library-size case + outcome model + viability governor + selector + directional 3D residual memory, tested across W0--W3 using repeated-launch history lengths 0, 5, 10, 20, 50, and 100 before any real repeated launches.
- Real flight: fixed-gate repeated-launch validation with Vicon-measured initial states.

- Writing protection: 08--15 June is protected for thesis writing and final integration.
