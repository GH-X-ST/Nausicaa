## Purpose

Use this guide for weekly progress reports and meeting summaries. The report should tell supervisors what changed, what evidence exists, what failed, and what decision is needed next.

## Structure

Recommended structure:

1. One-sentence project status.
2. Main technical progress.
3. Evidence obtained.
4. Problems or blocked claims.
5. Next-week plan.
6. Decision or feedback requested.

## Current mission language

Use:

```text
environment-conditioned primitive selection
local flow-context features
fixed-gate repeated launches
finite lift dwell
directional 3D residual lift belief
primitive outcome evidence
primitive-controller variant evidence
post-W3 library-size cross-study
viability-filtered primitive selection
sim-real replay pairing
environment-randomised robustness
```

Avoid:

```text
perching-like final objective
same-flight recapture
all-arena exploration
fan-layout-specific controller
large-angle agile turn as required final behaviour
```

## Weekly evidence standard

A good report should contain at least one of:

- a quantitative table;
- a plot or figure reference;
- a failure-label summary;
- a runtime/storage summary;
- a clear claim boundary;
- a real-flight readiness or risk note.

## Runtime/storage reporting

For any dense simulation week, report:

```text
row count
chunk count
worker count
largest file size
storage format
resume/repair status
file-size audit result
```

## Example compact weekly summary

```text
This week shifted the controller to environment-conditioned primitive selection. The main evidence is a chunked primitive archive with local flow-context fields and an initial outcome-model audit. The next decision is whether W3 environment randomisation should prioritise fan-position shifts or fan-power changes.
```

## Avoid AI-like progress writing

Poor:

```text
This week made substantial progress toward a robust and comprehensive framework.
```

Better:

```text
This week produced a 480k-row primitive outcome archive, a context-feature audit, and a blocked-claim note for real-flight transfer.
```
