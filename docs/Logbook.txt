## Purpose

Use this guide for daily logbook entries and progress records. A logbook is not thesis prose. It records what changed, what evidence exists, what failed, and what the next decision is.

## Style

Use compact bullet points. Preserve the existing LaTeX spacing style when writing TeX entries:

```latex
\vspace{0.05cm}
...
\vspace{0.30cm}
...
```

Do not write long literature-review paragraphs in the logbook.

## Daily entry structure

Each day should include:

- work completed;
- evidence or result obtained;
- failure or limitation found;
- implication for the next step.

For the current mission, classify each day as one of:

```text
method change
simulation evidence
real-flight evidence
instrumentation
writing / claim boundary
runtime / storage governance
```

## Current project wording

Use:

```text
environment-conditioned primitive selection
local flow-context features
fixed-gate repeated launch
directional 3D residual lift belief
finite lift dwell
primitive outcome evidence
primitive-controller variant evidence
post-W3 library-size cross-study
sim-real replay pairing
environment-randomised robustness
```

Avoid:

```text
perching objective
same-flight recapture
all-arena exploration claim
fan-layout-specific controller
mission success, unless actually shown
```

## Quantitative logbook rule

Whenever possible, include a number:

```text
row count
launch count
success count
failure count
minimum wall margin
lift dwell time
energy residual
runtime
worker count
file size
partition count
```

Do not record only that something 鈥渨orked鈥? State what was checked.

## Runtime and file-size entries

For dense runs, log:

```text
runner name
workers
chunk size
partition count
largest file size
storage format
resume / repair status
checksum status
```

Any file above 100 MB must be logged as a rule violation or explicitly approved local-only evidence.

## Claim boundary

Each major entry should include a blocked-claim sentence when relevant:

```text
This is simulation-only evidence and does not yet support a real-flight transfer claim.
```

or:

```text
The flight was usable for instrumentation, but not for controller comparison because tracking was lost before termination.
```
