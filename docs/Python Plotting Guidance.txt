# Python Plotting Guidance — Robotics-Journal Figures

## Purpose

Use this guide when creating plots for thesis, slides, logs, or robotics-style paper figures. Figures should show physical evidence, not decorative output.

## General style

Use:

- white background;
- clear axes and units;
- restrained colours;
- readable labels;
- deterministic export;
- captions that state the claim;
- high-resolution PNG for thesis figures.

Recommended export:

```python
fig, ax = plt.subplots(figsize=(5.2, 3.0), dpi=600)
fig.patch.set_facecolor("white")
ax.set_facecolor("white")
...
fig.savefig(out_path, dpi=600, facecolor="white", bbox_inches="tight")
plt.close(fig)
```

## File-size rule

Every generated plot file should remain below 100 MB. Prefer below 25 MB for thesis figures and below 75 MB for dense diagnostics. If a plot exceeds 100 MB, simplify it, rasterise dense layers, split it into panels, or move full data to compressed partitions.

## Figure types needed

### 1. Environment context map

Show:

- arena outline;
- local updraft field;
- context features such as wing-mean wind or spanwise gradient;
- fixed launch gate or representative current state;
- safety volume.

### 2. Primitive outcome map

Show accepted, weak, failed, and rejected primitive outcomes over physically meaningful axes:

For R6/R7 figures, distinguish primitive-local LQR variant diagnostics from contextual archive evidence. R6 figures should explain entry roles, Q/R tuning evidence, and primitive-controller variant selection. R7 figures should show archive coverage, clustering/library shrinkage, and outcome-model evidence for fixed primitive-controller variants.

```text
speed
heading error
wing-mean wind
left-right wind difference
wall margin
energy residual
lift dwell time
```

Also show `episode_terminal_useful` or `boundary_use_class == "episode_terminal_useful"` separately when x/y wall or lateral safety-volume exits are retained as terminal episode evidence. Do not colour these rows as continuation success.

### 3. Repeated-launch episode traces

Show multiple episodes with:

- trajectory path;
- accepted primitive;
- termination point;
- local updraft contour;
- belief before/after where useful.

Do not overlay too many episodes in one plot.

### 4. Belief evolution

Show:

- belief before and after repeated launches;
- observed lift evidence;
- decay setting;
- capture or dwell metric over launch index.

### 5. Sim-real pairing

Show predicted versus measured:

- energy residual;
- lift dwell time;
- capture success;
- termination cause;
- minimum wall margin.

### 6. Generalisation test

Show validation-layer and environment-instance effects without turning them into online branches:

- shifted fan position;
- changed fan power;
- active fan subset;
- randomised updraft amplitude or width.

Use W0 as dry air only, W1 as Gaussian plume only, W2 as GP-corrected annular-Gaussian only, and W3 as randomised GP-corrected annular-Gaussian only.

## Colour guidance

Use consistent semantic colours:

```text
accepted / safe: green
weak / marginal: amber
failed / unsafe: red
rejected before execution: grey
baseline: blue or dark grey
context-conditioned controller: green or purple
```

Do not use rainbow maps for categorical outcomes.

## Label rules

Every axis must include units where applicable:

```text
x_w (m)
y_w (m)
z_w (m)
speed (m s^{-1})
yaw error (deg)
energy residual (m)
lift dwell time (s)
minimum wall margin (m)
wing-mean vertical wind (m s^{-1})
```

## Caption standard

Good caption:

```text
Primitive outcome map under measured-updraft replay. The accepted region expands with wing-mean vertical wind, while low-speed and low-margin cases remain rejected by the viability governor.
```

Poor caption:

```text
Results of simulation and real flight.
```

## Audit metadata

Each saved plot should have a nearby source note or manifest stating:

```text
input files
filtering rules
sample count
environment_id
latency case
policy name
created date or run ID
file size
```
